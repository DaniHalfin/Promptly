# Promptly Spec — Changelog

> **Reference file.** This document tracks the revision history of the Promptly product spec.

### v1.6 (2026-06-22)
- **[OQ-1 RESOLVED] OpenAI per-model cost approximation:** Token-fraction approximation accepted for MVP. Per-model OpenAI cost is described as "estimated via token-fraction approximation; exact per-model billing data not available from the OpenAI API" in §5 Source 2 (Assumptions) and §7.6 (Model cost share Note). OQ-1 marked resolved in §13.
- **[OQ-2 RESOLVED] GitHub Copilot OAuth scopes:** Classic PAT required (fine-grained PATs not supported by the billing usage endpoints). Org endpoint (`/organizations/{org}/settings/billing/ai_credit/usage`): requires `repo` scope (recommended for onboarding UI) or `admin:org` scope; user must be org admin (owner) or billing manager. Individual user endpoint (`/users/{username}/settings/billing/ai_credit/usage`): requires `user` scope; user must have a self-purchased Copilot plan (Free/Pro/Pro+/Max) — org/enterprise-licensed users must use the org endpoint. API version header `X-GitHub-Api-Version: 2026-03-10` required. Data retained for 24 months. §5 Source 4 Assumptions updated with exact token type and scope requirements. OQ-2 marked resolved in §13.
- **[OQ-10 RESOLVED] Claude Code session data format:** `~/.claude/stats-cache.json` does not exist. Correct data location is `~/.claude/projects/<encoded-project-path>/<sessionId>.jsonl` (or `$CLAUDE_CONFIG_DIR/projects/...` if `CLAUDE_CONFIG_DIR` is set). Format is JSONL session transcripts with per-message token counts. Cost is not stored in the files — computed as tokens × model price via the LiteLLM price map. §5 Source 1 (Files read, Data returned, Assumptions, Error states), §4 (Step 2 connection table, Step 3 validation), §11 (adapter comment, local files diagram), and the Danny persona updated. OQ-10 marked resolved in §13.
- **[BLOCKING COUNT] 0 blocking OQs remain.** OQ-1, OQ-2, and OQ-10 were the only blocking questions; all three are now resolved.

### v1.3 (2026-06-19)
- **GitHub Copilot reclassified: Tier C → Tier B.** Copilot Business and Enterprise moved to usage-based billing (AI credits; 1 credit = $0.01 USD) on June 1, 2026 (GA). The adapter now calls `GET /organizations/{org}/settings/billing/ai_credit/usage` for org users and `GET /users/{username}/settings/billing/ai_credit/usage` for individual plan users. Model attribution is now available (15+ models including Claude Sonnet 4.6, GPT-5.4, Gemini 3.5 Flash, with widely varying pricing per model). The dead `/orgs/{org}/copilot/usage` endpoint (shut down April 2, 2026) is removed from all references.
- **Copilot adapter renamed:** `github.js` → `github_copilot.js` to match the `github_copilot` source_id and align with all other adapter filenames.
- **Copilot metrics updated:** Old Tier C subscription-value metrics (acceptance-rate-only, lines-to-token heuristic, cost-per-accepted-suggestion, relative subscription comparison) replaced with Tier B cost-denominated metrics: total AI credit spend (gross/net), spend breakdown by model, cost per interaction, model distribution. Acceptance rate retained from engagement metrics reports (not cost-denominated).
- **Recommendation engine updated:** R2 (Downgrade Model) now applies to GitHub Copilot Chat/CLI, with a Copilot-specific model downgrade note added. All four recommendations updated to accurately reflect which data Copilot can contribute.
- **Tier C:** No P0 MVP sources remain at Tier C. Tier C is retained as a defined placeholder for future P1 sources (ChatGPT export, Claude.ai export, Cursor) when they ship.
- **Consistency fixes applied:** Removed duplicate YAML frontmatter block; §4 Step 1 source count corrected (4, not 5); §5 section header "(ship with v1.0)" removed; §1 Overview cost-framing updated to reflect all MVP sources have actual billing data; §11 architecture diagram frontend memory description corrected (removed "uploaded file contents"); OQ-2 updated to reflect new billing API; OQ-3 closed (subscription cost input no longer needed — actual billing data is available); OQ-5 closed (option (b) implemented per §9); OQ-6 gap noted (deliberately removed in v1.2; tombstone entry added to §13); document status set to Draft (three blocking OQs remain (OQ-1, OQ-2, OQ-10)).

### v1.2 (2026-06-19)
- **Data Sources — Full rewrite:** Reframed around the tool the user interacts with, not platforms or middleware. Added Claude Code (Anthropic CLI) as a P0 MVP source; moved ChatGPT and Claude.ai file exports to P1 (future releases). Added design principle: Promptly is a *reader, not a collector*. All P0 sources use local file reads or one-time API calls — no background processes, no ongoing dependencies.
- **Recommendation Engine:** Locked to 4 computable recommendations (no prompt content required): Enable Prompt Caching (R1), Downgrade Model (R2), Reduce Prompt Verbosity (R3), Use Off-Peak Hours (R4). Deferred Context Length Accumulation, Quota Risk, and Copilot Subscription Value to future scope.
- **Out of Scope expanded:** Explicitly excludes LiteLLM and Helicone as P0 sources (continuous stores, not one-time reads), flat-subscription web users (ChatGPT Plus, Claude.ai Pro), prompt content analysis, and background data collection or agent installation.
- **File upload references removed:** Promptly reads data directly from tools' local files or APIs. P1 file-export sources will use local path reads when implemented — users do not upload anything manually.
- **Version:** Bumped from 1.0 to 1.2.
