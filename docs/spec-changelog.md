### v2.1 (2026-07-05)
- **Analysis page redesigned as narrative analyst report:** The results page is restructured from a billing-statement/source-panel layout to a narrative analyst-report flow. Fixed section order: Analysis header → Where is your money going? (spend-by-tool bar) → Is your spending trending? (cross-source trend chart) → Per-tool cards → Budget tracker CTA → Export actions. Per-source panels removed; per-tool expandable cards replace them.
- **Uniform "Estimated spend" label:** All user-facing spend totals are labeled "Estimated spend" across the entire UI. "Estimated" is a conservative product label — some sources provide precise billing data, others compute cost locally. Source-level accuracy documented in README and export caveats. §7.1 renamed accordingly.
- **New cross-source metrics (§7.3a–§7.3e):** Effective cost per million tokens; cross-source daily spend roll-up; spend by tool (share); explicit trend status object (available / insufficient\_data / no\_prior\_spend — never silently hidden); spike callout (peak day ≥ 2× average AND ≥3 days of data, with threshold rationale documented).
- **Analysis period mandatory:** Must appear in page header at all times; never hidden.
- **Token sub-line format:** `X.XM tokens · $Y.YY per million tokens` as visible inline text (accessibility requirement — no tooltip dependency). §7.2 and §7.3a.
- **Model cost share:** Chart type changed from pie to horizontal bar. Dominant model sentence added: "Most of your [source] spend went to [Model X] ([Y]%)." §7.6.
- **Input/output signal:** Renamed §7.7 from raw ratio to plain-language efficiency signal (`input_heavy` / `output_heavy` / `balanced`). Raw ratio retained for R3 trigger and JSON export only.
- **Cache display:** §7.8 renamed from "Cached token fraction" to "Cache savings." Analysis-page display changed from cache hit rate % to dollar savings: "You saved ~$Y this period." Cache fractions retained internally for R1 trigger and export.
- **Average session cost (§7.18):** New metric. Claude Code and Copilot: "Each session costs you on average $X." OpenAI and Anthropic: always display "Session-level cost not available from provider data" — never omitted, never substituted.
- **Recommendation placement:** Recommendations no longer appear in a global list. Top 1–2 money-saving recommendations (with specific dollar savings) appear under the spend-by-tool bar. Per-tool recommendations appear at the bottom of the relevant tool card. Generic frontend-generated advice prohibited.
- **R1:** Clarified as opportunity-framed (projected savings going forward), not realized savings.
- **R2:** Estimated savings requirement made explicit: R2 must include a specific dollar saving or be suppressed. Copilot downgrade map confirmed: `claude-fable-5` → `claude-haiku-4.5`.
- **R3:** Added conservative approximate savings estimate: excess input reduction of ~20%, priced at source/model input rates. Labeled as approximate in UI. R3 appears in the relevant tool card.
- **R4 removed from scope:** R4 (off-peak/latency) is a throughput optimization with no cost impact and is applicable only to Claude Code. Deferred to a future "performance insights" feature. R4 must not appear in any output.
- **Export mirrors narrative layout:** §10 PDF structure follows header → spend breakdown → trend → per-tool cards → recommendations order. All spend labels use "Estimated spend." Top-3 recommendation cap removed (show all triggered). Assumptions/caveats section documents per-source accuracy and estimate basis. JSON schema adds new cross-source narrative fields.
- **§9 Privacy:** Added note that narrative signals (efficiency signal, session cost, cache savings) are computed from aggregated metrics only; no prompt or response content is read to generate them.
- **§11 Architecture:** Pie chart removed as required chart type. Horizontal CSS bars and Recharts bar charts are the primary pattern. Backend presentation/metric layer added. Narrative results component hierarchy documented.
- **§12 Out of Scope:** Budget tracker explicitly noted as CTA placeholder only (no implementation). R4 latency optimization recommendations noted as deferred.
- **§13 Open Questions:** OQ-11 through OQ-15 added and resolved (Q1–Q4, Q8). OQ-16 through OQ-18 added as deferred ED/code-level items.

### v1.2 (2026-06-19)
- **Data Sources — Full rewrite:** Reframed around the tool the user interacts with, not platforms or middleware. Added Claude Code (Anthropic CLI) as a P0 MVP source; moved ChatGPT and Claude.ai file exports to P1 (future releases). Added design principle: Promptly is a *reader, not a collector*. All P0 sources use local file reads or one-time API calls — no background processes, no ongoing dependencies.
- **Recommendation Engine:** Locked to 4 computable recommendations (no prompt content required): Enable Prompt Caching (R1), Downgrade Model (R2), Reduce Prompt Verbosity (R3), Use Off-Peak Hours (R4). Deferred Context Length Accumulation, Quota Risk, and Copilot Subscription Value to future scope.
- **Out of Scope expanded:** Explicitly excludes LiteLLM and Helicone as P0 sources (continuous stores, not one-time reads), flat-subscription web users (ChatGPT Plus, Claude.ai Pro), prompt content analysis, and background data collection or agent installation.
- **File upload references removed:** Promptly reads data directly from tools' local files or APIs. P1 file-export sources will use local path reads when implemented — users do not upload anything manually.
- **Version:** Bumped from 1.0 to 1.2.
